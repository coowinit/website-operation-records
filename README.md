# 网站运维工作记录表

版本：`v1.0.0`

这是一个用于记录网站运维工作的静态 HTML 页面。它适合放在公司内部共享目录、GitHub 仓库或云盘中，用于证明网站安全巡检、备份、询盘排查、网站更新和 SEO 巡检整改等工作已经建立记录，并且可以持续补充。

## 项目目标

本项目重点不是做复杂系统，而是建立一套简单、稳定、可长期维护的工作记录页面。

当前包含 5 张表：

1. 网站安全巡检表
2. 网站备份清单
3. 询盘链路异常排查清单
4. 核心网站更新留痕表
5. SEO巡检及整改记录表

所有详细材料统一通过 **“云文件”** 字段跳转查看。页面中只保留基础索引信息，具体截图、报告、排查过程、整改说明等内容建议放在云文件中。

## 当前版本特点

- 一个 `index.html` 页面集中展示 5 张表；
- 页面上方使用 5 个菜单按钮切换表格；
- 菜单一行 3 列，保持轻量展示；
- 页面主体宽度为 `1400px`，适合展示表格内容；
- 表格内容由 `js/records-data.js` 统一管理；
- 页面渲染、本地保存、导入导出由 `js/app.js` 负责；
- 支持新增、编辑、删除记录；
- 支持导出 JSON 备份；
- 支持导入 JSON 备份；
- 支持恢复默认模板。

## 日常使用方式

直接双击打开：

```text
index.html
```

打开页面后可以进行：

- 新增记录；
- 编辑记录；
- 删除记录；
- 导出备份；
- 导入备份；
- 恢复默认模板。

## 重要提醒

页面上的新增、编辑、删除会保存到当前浏览器本地，不会自动写回源代码。

因此建议：

1. 每次完成重要修改后，点击 **“导出备份”** 保存一份 JSON 文件；
2. 换电脑、换浏览器或清理浏览器数据后，如需恢复记录，使用 **“导入备份”**；
3. 如果要把数据固定到项目版本中，可以把备份 JSON 中的记录整理回 `js/records-data.js`。

## 数据维护文件

主要维护文件：

```text
js/records-data.js
```

每张表由以下结构组成：

```js
window.recordsData = {
  table01: {
    title: "网站安全巡检表",
    menuTitle: "网站安全巡检表",
    description: "表格说明",
    note: "使用说明",
    columns: [
      { key: "recordId", label: "记录编号" },
      { key: "date", label: "巡检日期" },
      { key: "site", label: "网站名称" },
      { key: "cloudFile", label: "云文件", type: "link" }
    ],
    rows: [
      {
        recordId: "SEC-2026-07-001",
        date: "2026-07-__",
        site: "英文官网",
        cloudFile: { text: "查看详情", url: "https://你的云文件链接" }
      }
    ]
  }
};
```

## 新增记录

可以直接在页面中点击 **“新增记录”**。

也可以在 `records-data.js` 中对应表格的 `rows` 里新增一条数据：

```js
{
  recordId: "SEC-2026-07-002",
  date: "2026-07-__",
  site: "英文官网",
  cloudFile: { text: "查看详情", url: "https://你的云文件链接" }
}
```

## 新增、删除或调整列

### 新增列

在对应表格的 `columns` 中增加一项：

```js
{ key: "owner", label: "负责人" }
```

然后在需要的记录中补充：

```js
owner: "Winson"
```

### 删除列

只需要从 `columns` 中删除对应字段即可。

`rows` 中即使保留旧字段，页面也不会显示。

### 调整列顺序

直接调整 `columns` 数组的顺序即可，不需要重写每一行数据。

## 新增或删除表格

### 新增表格

在 `records-data.js` 中复制一份 `table05`，改成 `table06`，再修改：

- `title`
- `menuTitle`
- `description`
- `note`
- `columns`
- `rows`

页面会自动生成新的菜单按钮。

### 删除表格

从 `records-data.js` 中删除对应的 `tableXX` 即可。

## 云文件链接格式

所有详细说明、截图、报告、排查过程和整改记录都建议放在云文件中。页面中只保留 “查看详情” 入口。

```js
cloudFile: {
  text: "查看详情",
  url: "https://你的云文件链接"
}
```

## 导入、导出与备份存放建议

页面支持通过 JSON 文件进行数据备份和恢复。由于页面中的新增、编辑、删除记录默认保存到当前浏览器本地，所以 **导出备份是非常重要的操作**。

### 导出备份

点击页面中的 **“导出备份”**，会生成一个 JSON 文件，例如：

```text
website-work-record-backup-202607081130.json
```

建议每次完成重要记录更新后，都导出一份备份文件。

### 备份文件建议放哪里

日常导出的 JSON 备份文件，建议统一放到项目中的 `backups/` 目录：

```text
website-work-record-v1.0.0/
├── backups/
│   ├── 2026-07-08-operation-records.json
│   ├── 2026-07-15-operation-records.json
│   └── 2026-07-31-operation-records.json
```

推荐命名格式：

```text
YYYY-MM-DD-operation-records.json
```

如果一天导出多次，可以加上时间：

```text
YYYY-MM-DD-HHMM-operation-records.json
```

例如：

```text
2026-07-08-1530-operation-records.json
```

### 正式版本压缩包放哪里

项目正式版本的压缩包，例如：

```text
website-work-record-v1.0.0.zip
```

建议放到 GitHub 的 **Release 附件** 中，不一定放到项目目录里。

这样可以区分：

| 文件类型 | 建议位置 | 说明 |
|---|---|---|
| 日常 JSON 数据备份 | `backups/` | 用于恢复页面记录数据 |
| 正式版本 ZIP 压缩包 | GitHub Release 附件 | 用于归档完整版本 |
| 页面源代码 | 仓库主目录 | 用于维护系统页面 |
| 默认模板数据 | `js/records-data.js` | 用于初始化表格结构和默认记录 |

### 公开仓库注意事项

如果仓库是公开的，真实工作记录、内部网站名称、云文件链接、服务器信息等内容不建议直接上传。

可以采用这种方式：

```text
backups/
├── example-operation-records.json    # 示例备份，可以上传
└── private/                          # 真实备份，建议本地或云盘保存，不上传
```

如果是公司内部仓库或私人仓库，可以把真实 JSON 备份文件直接放到 `backups/` 目录中。

### 导入备份

点击页面中的 **“导入备份”**，选择之前导出的 JSON 文件即可恢复数据。

导入后，数据会保存到当前浏览器本地。建议导入确认无误后，再重新导出一份最新备份。
